Section 01

Current Page Audit

Reading the actual page.tsx and CSS against the three questions every homepage must answer within two seconds: What is this? Who is it for? What do I do next?

✓ Working

Dark aesthetic & typography

Syne display + Outfit body + JetBrains Mono is a distinctive and credible stack. The surface/brand/amber color palette is uncommon and memorable. This is a real differentiator — don't change it.

✓ Working

Real numbers with credibility

Stats bar with 0.045ms, 64%, 37%, $9/mo are concrete. Specificity creates trust. The feature cards with measured stats ("64% hit rate in production") signal this is a real product, not vaporware.

✓ Working

R3F hero scene

The Three.js background is visually impressive and technically differentiates this page from generic SaaS. Keeping this is correct. It communicates "this is technical infrastructure" without a word.

✓ Working

How It Works — 4 steps

The numbered steps section is clean, scannable, and informative. Good structure. Needs stronger visual treatment but the copy and logic are sound.

✗ Broken

Split hero CTAs — wrong primary action

"Read the Docs" is not a conversion CTA. It's a research CTA. Sending visitors to docs before they're sold means they leave. The primary button in the hero must drive toward sign-up / pricing — not education.

✗ Broken

Zero social proof anywhere

No testimonials. No customer logos. No usage numbers. No "used by X developers." This is the single biggest conversion killer on the page. Trust signals outperform design improvements every time.

✗ Broken

Technical jargon before the emotional hook

"Cryptographic Merkle proofs. Markov-chain prediction." appears before anyone understands what problem is solved. The visitor must first feel the pain, then receive the solution. The current page reverses this order.

✗ Broken

Waitlist when product is live

"Stay in the loop" with a waitlist form when you're already selling at $9/mo confuses visitors. It signals the product isn't ready. Replace with a direct "Get Your Instance" CTA section.

⚠ Weak

Hero answers only 1 of 3 questions

What is this? ✓ "Persistent, Verifiable Memory for AI" is clear. Who is it for? ✗ Not clear — developers? agents? enterprises? What next? ✗ Two equal buttons create decision paralysis.

⚠ Weak

No emotional story — leads with architecture

The marketing arsenal has extraordinary lines ("Every AI conversation starts from zero," "A second brain for everyone") that aren't used on the page. The page leads with what MMPM is, not why someone should care.

⚠ Weak

Fragmented CTA journey — 5 different actions

Docs → Watch Live → Open Visualiser → Waitlist → View Plans. Each CTA competes. Visitors are pulled in 5 directions. A visitor who wants to sign up has to figure out where to go on their own.

⚠ Weak

Who it's for is never stated

The target customer (AI developers building agents with Claude/GPT) is never named. "Dedicated AI memory instances" alone doesn't tell a developer whether this is for their use case. One line of positioning would help enormously.

Section 02

5 Principles Applied

Each principle mapped to specific changes for this page.

01

Prioritise Clarity

Answer 3 questions in 2 seconds

The hero answers "what is this" but fails "who is it for" and splits "what next" across two equal buttons.
New hero subtitle: "Built for AI developers building with Claude, GPT, and MCP-compatible agents." Add one clear primary CTA: "Get Your Instance — from $9/mo →"

The tagline "Stop re-explaining. Start remembering." hits the pain immediately, then the subtitle explains the solution.

02

Singular CTA

One path. Repeated consistently.

5 different CTA destinations across the page. Docs, Visualise, Visualise again, Waitlist form, Pricing. Visitors are lost.
Primary CTA: "Get Your Instance →" → /pricing on every section. Secondary: "See It Live" → /visualise, styled smaller and less prominent.

The primary CTA appears in hero, after stats, after social proof, and at the bottom. Every section points to the same destination.

03

Social Proof

Trust before claims

Zero testimonials, zero logos, zero usage numbers. A visitor has no external validation that this product works or is used by real people.
Add: usage counter ("847 atoms stored today"), developer quote(s), a "We eat our own cooking" section showing MMPM runs on MMPM. This is the killer line from the arsenal.

Even one genuine testimonial from a real user outperforms a full design overhaul. The "dogfooding" story is uniquely credible.

04

Intentional Minimalism

Let the product breathe

The existing page is actually close to this principle. The R3F scene, clean cards, and generous whitespace are good. The problem is content density in the wrong places.
Remove the waitlist section (it clutters the conversion path). Compress the pricing CTA section. Add breathing room above and below the hero text. Let silence do the work.

Restraint: no new animations needed. The R3F hero IS the animation. Everything else should be still and confident.

05

Micro Details

Small details build trust

Feature icons are generic SVGs. The waitlist form has no contextual help text (just "No spam"). The step numbers are styled large but add no interactivity.
Custom icons for each capability (Merkle shield, Markov arc, lightning bolt, plug). Add helpful microcopy near the sign-up: "Your instance is ready in under 60 seconds." Add a copy-to-clipboard on the MCP endpoint example.

The "60 seconds to ready" detail (from the business plan) makes the product feel fast and professional. Small trust signals compound.

Section 03

The Compelling Story Arc

The page must tell a story that moves visitors from awareness to conviction. Use the Pain → Agitate → Solution → Proof → Social → CTA structure (PAS+). Every line comes from your Marketing Arsenal.

1

PAIN — The Universal Problem

"Every AI conversation starts from zero."

Open with the pain. Your visitor already knows this feeling — they've pasted the same context 50 times. Don't explain MMPM yet. Just name the problem so clearly they feel seen.

Placement: Above the fold hero badge or pre-headline line.

2

AGITATE — Make the pain concrete

"You re-explain your preferences. Your agent forgets your decisions. You paste the same context. Every. Single. Session."

Amplify the frustration before offering the solution. This is where visitors think "yes, this is exactly my problem." Can be rendered as a subtle animated "before" state — a chat window resetting to zero.

Placement: First scroll section, or integrated into the hero subtitle.

3

SOLUTION — The positioning statement

"Parametric Memory gives your AI a second brain. Persistent. Verified. Yours alone."

Now introduce MMPM. Simple, clean, three words that matter. Don't mention Merkle proofs yet — just deliver the outcome. The technology proof comes in the next step.

Placement: H1 headline. Use the gradient text treatment you already have.

4

PROOF — Technical credibility

"Verified by math, not trusted on faith. RFC 6962 Merkle proofs. 64% Markov hit rate. 0.045ms recall. Your own substrate — not a shared database."

Now the technical depth lands because the visitor is already convinced they need this. The stats feel like proof, not complexity. This is where your existing stats bar, feature cards, and how-it-works section belong.

Placement: Stats bar → Features → Architecture — all existing sections, just reordered with the story above them.

5

SOCIAL — We trust it with ours

"We don't just sell you a second brain — we trust it with ours. Every billing event, health check, and customer signup at Parametric Memory runs on MMPM."

The killer line from your arsenal. This is your social proof. You are your own first customer, your own best case study. This is genuinely rare and credible — especially for a technical audience.

Placement: New section between features and pricing. Can include a live atom counter (hook to your API).

6

CTA — One clear action

"Get your dedicated instance. Ready in 60 seconds. Starting at $9/month."

After the full story is told, the CTA lands like a natural conclusion rather than a sales pitch. The specifics (60 seconds, $9/mo) reduce friction by answering the two questions all visitors have before clicking.

Placement: Final CTA section, replacing the current fragmented waitlist + pricing CTA.

Section 06

Social Proof Plan

You don't need external customers to start. Your own product running on MMPM is the most credible proof story in the arsenal. Here's how to build the social proof layer progressively.

🔐

Dogfooding Story (Right Now)

Use the killer line from the arsenal: "We trust it with ours." Create a section explaining that Parametric Memory's own billing, health monitoring, and customer ops all run on MMPM. Add a live atom counter from your actual substrate. This is uniquely honest and credible.

📊

Usage Numbers (Right Now)

Even your own usage numbers work: "821 atoms stored," "4 Merkle trees active," "64% hit rate in production." Real numbers from the live visualise page can be pulled via API and shown as live trust signals on the landing page.

💬

First User Testimonials (Next 30 Days)

As early users come in, ask for a quote. Even one genuine quote from a developer ("I stopped re-pasting my architecture docs into Claude") beats a polished design. Leave a placeholder section ready so it's easy to add.

📅

Production Timeline (Right Now)

"In production since March 2026" — a simple timestamp signals maturity and that this isn't vaporware. Combined with the atom count and hit rate stats, it tells a credible launch story.

🔗

Integration Logos (When Available)

As you build integrations and partnerships, add the logo bar: "Works with Claude · OpenAI · Cursor · Windsurf · Any MCP client." Even MCP logos (which are public spec) add legitimacy. Claude's logo with a "works with" tagline is powerful.

📰

Show HN / Press (Post-Launch)

Reserve a spot for "As featured on Hacker News" or the HN upvote count post-launch. A Show HN post that gets 200+ upvotes becomes permanent social proof. Plan the section now so it's easy to activate.

Section 08

Remotion Visual Concept

Remotion is the right tool for creating a short product explainer video (15–25s) that shows the "before and after" of AI memory. This renders as a video file that can be embedded in the landing page as an MP4 loop — no runtime Three.js cost, just a video tag.

Why Remotion, not more R3F on the page?

The R3F hero scene handles the "impressive technical visual" role. Remotion handles the "product story" role — showing the actual problem being solved in a concrete, relatable way. A 20-second video that shows an AI agent forgetting and then remembering is worth more than another 3D scene.

📹 Proposed Video — "Before & After Memory" (20 seconds · 1920×1080 · 30fps)
010–5s

The Problem — AI Amnesia

Split screen. Left: AI chat window. User types "My name is Alex, I prefer TypeScript, no semicolons." Right: same chat, next day. User types "Good morning." AI responds: "Hello! How can I help you today?" — no memory. A red ✗ appears.

Frame 0–150 · Text + chat UI animation · fade-in/out with spring easing
025–10s

The Solution — MMPM Connected

The MMPM logomark appears between the user and the AI. An atom (glowing particle) flows from the conversation into the memory substrate — visually showing storage. The Merkle tree assembles one node. Root hash glows amber.

Frame 150–300 · Particle physics + Merkle node build · spring() animations
0310–16s

Memory Recall — Next Session

New chat session starts. The Markov arc fires — a prediction arc lights up before the user types. User types "Good morning." AI responds: "Good morning, Alex! Ready to continue our TypeScript project? I remember you prefer no semicolons." A green ✓ appears.

Frame 300–480 · Arc animation + text reveal + checkmark
0416–20s

The Close — Logo + CTA

Camera pulls back to reveal the full Merkle substrate — dozens of glowing atoms, connected by Markov arcs, all nested in the tree structure. Parametric Memory wordmark fades in. Tagline: "Stop re-explaining. Start remembering." Fade to black.

Frame 480–600 · Reveal + wordmark + tagline · interpolate opacity

Technical Notes for Remotion Build

Uses @remotion/three for the Merkle tree 3D scene (reuse existing Three.js geometry from the R3F hero). Text animations with spring() from frame. TransitionSeries with fade() for scene transitions. Render to MP4 at 30fps, then compress with ffmpeg to <5MB. Embed on landing page with <video autoplay muted loop playsinline>.

Alternative: Interactive inline demo (no Remotion needed)

If Remotion setup is too heavyweight for this sprint, a simpler path: use CSS animations to create the "before/after" chat mockup directly in the landing page. Two chat bubbles, a MMPM connector icon, and counter-up stats. Ships faster, achieves 80% of the impact. Remotion is the premium path for a proper launch video.
